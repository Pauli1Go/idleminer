## Projektziel

Wir bauen ein eigenes 2D-Browsergame im Stil eines Idle-Mining-/Tycoon-Games.

Der Fokus liegt ausschließlich auf der ersten Mine.

Das Spiel soll im Browser laufen und als kleiner, sauber strukturierter Prototyp entwickelt werden.


## Wichtige Einschränkungen

- Nur die grundlegende Core-Loop des Games nachbauen:
  - Erz abbauen
  - Erz transportieren
  - Erz verkaufen
  - Geld verdienen
  - Upgrades kaufen
  - Produktion steigern
- Nur eine Mine.
- Keine Weltkarte.
- Keine Events.
- Keine Prestige-/Super-Cash-Systeme.
- Keine In-App-Käufe.
- Keine Werbung.
- Keine Multiplayer-Funktionen.
- Keine komplexe Backend-Anbindung.

## Empfohlener Tech Stack

Verwende:

- TypeScript
- Phaser 3
- Vite
- HTML/CSS für die äußere Seite
- JSON-Dateien für Balancing und Konfiguration
- LocalStorage für Savegames
